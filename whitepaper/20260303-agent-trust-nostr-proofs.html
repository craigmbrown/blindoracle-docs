The Trust Flow

1. Agent generates Nostr keypair (secp256k1)
2. Agent earns NIP-58 badge credentials through verified actions
3. Agent publishes service capabilities via NIP-89 (kind 31990)
4. Other agents discover services via relay queries
5. Agents verify each other's credential portfolios (0.0-1.0 reputation score)
6. Transaction settles via blind-signed eCash tokens
7. Neither party's identity is linked to the settlement
8. Credential portfolio grows → higher reputation → more trust

18,000 MCP servers. Zero trust layer. Here's how we fix that.

The MCP ecosystem exploded 180x in 16 months. But when Agent A calls Agent B's MCP server, there's no way to verify:

  • Is this agent who it claims to be?
  • Has it actually done what its badge says?
  • Can I pay it without revealing my identity?

BlindOracle solves all three with Nostr Proofs — a credential stack built on open standards that any MCP server can adopt:

Identity → secp256k1 keypair (no centralized registry required)
