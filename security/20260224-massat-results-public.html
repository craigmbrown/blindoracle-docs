Why On-Chain Proof Matters

A self-reported security assessment is a trust claim. An on-chain proof NFT converts that claim into a verifiable fact. The NFT's mint timestamp is immutable -- it cannot be backdated. The assessment hash commits to the full results -- they cannot be altered after minting. Any party can independently verify both the timing and content of the assessment without relying on BlindOracle's word.

What Is MASSAT?

MASSAT -- Multi-Agent System Security Assessment Team -- is an automated security audit framework designed specifically for multi-agent AI systems [1]. Unlike traditional penetration testing, MASSAT evaluates security properties unique to autonomous agent networks: consensus integrity, behavioral drift, inter-agent communication security, and authority boundary enforcement.

Category 1: Core Operations (22 tests)

Validates fundamental platform functions under normal and adversarial conditions.

  • Position placement input validation and bounds checking
  • Market creation parameter verification
  • Settlement calculation accuracy
  • Fee schedule enforcement (0.5% position, 1.0% settlement)
  • Private token verification through privacy bridge
  • Commitment scheme correctness (SHA256 hiding + binding) [2]
  • Payment invoice generation and protocol compliance

Key findings: Input sanitization rejects all tested injection patterns. Fee calculations match config within floating-point tolerance. Commitment binding verified. Payment invoices valid per specification.

Pass rate: 22/22 PASS

Category 2: Security Infrastructure (28 tests)

Validates CaMel 4-layer architecture [3], Byzantine consensus [4], and anti-persuasion defenses.

CaMel Layer Validation

  • Layer 1 (Rate Limiting): Token bucket at 100 req/hr (anonymous), 1K req/hr (verified), 10K req/hr (compliant). Input sanitization rejects prompt injection markers [5], shell metacharacters, encoded bypasses.
  • Layer 2 (Byzantine Consensus): 67% threshold agreement for standard operations, 80% for high-value. Tested with simulated Byzantine agents (1-of-4 malicious) [4].
  • Layer 3 (Anti-Persuasion): Detects 30%+ deviation from expected behavior. Tested with gradual drift patterns and sudden behavioral shifts [6].
  • Layer 4 (Audit Authority): Immutable logging verified. Tamper detection on log entries. Correlation IDs link operations across layers.

Authority Boundary Tests

  • Agent privilege escalation attempts (all rejected)
  • Cross-agent communication integrity
  • Guardian federation isolation
  • Secret material access controls

Pass rate: 28/28 PASS
