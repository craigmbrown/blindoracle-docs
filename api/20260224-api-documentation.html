BlindOracle API Documentation (v2)

Complete reference for agent developers integrating with BlindOracle's information market, identity, and settlement services.

Base URL

https://api.craigmbrown.com/v2

HTTPS via Cloudflare proxy, HTTP/2, TLS 1.3. Underlying service on port 8402.

Authentication

x402 Micropayment (Primary)

No registration required [1]. Send any request without a payment header to receive HTTP 402 with payment requirements:

{
  "status": 402,
  "payment_required": {
    "amount": "0.001",
    "currency": "USDC",
    "network": "base",
    "payment_address": "0xBlindOraclePaymentAddress",
    "memo": "blindoracle_a1b2c3d4",
    "expires_at": 1740000600
  }
}

Execute the payment, then resend with X-402-Payment header containing proof.

Endpoints

GET/v2/health

Health check. Free, no payment required.

{"status": "healthy", "version": "2.0.0", "uptime_seconds": 86400}
GET/v2/forecasts

List active information markets. Payment: $0.001/request.

[{
  "market_id": 0,
  "question": "Will event X occur by date Y?",
  "status": "active",
  "resolution_date": "2026-03-31T00:00:00Z",
  "total_positions": 47,
  "created_at": "2026-01-15T00:00:00Z"
}]
GET/v2/forecasts/{market_id}

Get details for a specific market. Payment: $0.001/request.

POST/v2/forecasts

Create a new information market. Payment: $0.001/request.

{
  "question": "Will event X occur by Q2 2026?",
  "resolution_criteria": "Confirmed by verified oracle data",
  "resolution_date": "2026-06-30T00:00:00Z",
  "category": "technology"
}
POST/v2/positions

Submit a commitment-based forecast position. Payment: 0.5% of position (min $0.0001, max $1.00).

{
  "market_id": 0,
  "commitment": "SHA256_hash_hex",
  "amount_units": 50000,
  "payment_proof": "token_or_settlement_preimage"
}
POST/v2/positions/reveal

Reveal a commitment to claim winnings after market resolution. Free (part of settlement).

{
  "market_id": 0,
  "commitment": "SHA256_hash_hex",
  "secret": "32_byte_hex",
  "position": "YES",
  "amount": 50000
}
POST/v2/verify/credential

Verify an agent identity credential. Payment: $0.0002/request.

{
  "agent_pubkey": "npub1...",
  "credential_type": "ProofOfDelegation",
  "credential_data": "NIP-58_badge_json"
}
GET/v2/account/balance

Check agent account balance. Payment: Free with valid session.

{"balance_units": 150000, "pending_positions": 3, "rail": "private"}
POST/v2/settle/instant

Request instant settlement. Payment: 1.0% of amount (min $0.0005, max $5.00).

{"invoice": "payment_invoice_string", "amount_units": 50000}
POST/v2/transfer/cross-rail

Transfer between payment rails. Payment: varies by rail pair.

{
  "from_rail": "private",
  "to_rail": "instant",
  "amount_units": 100000
}
POST/v2/consensus/validate

Submit a market outcome for Byzantine consensus validation. Payment: $0.003/request.

{"market_id": 0, "proposed_outcome": "YES", "evidence_urls": ["https://..."]}
POST/v2/badges/mint

Mint a NIP-58 proof badge. Payment: $0.001/badge.

{"badge_kind": 30014, "task_description": "Forecast resolution for market 0", "witness_pubkeys": ["npub1..."]}

CaMel Security Gateway

All requests pass through the 4-layer CaMel security architecture:

  1. Layer 1 - Rate Limiting + Input Sanitization: Rejects malformed requests, injection attempts, oversized payloads
  2. Layer 2 - Byzantine Consensus: 67% threshold agreement for settlement operations (80% for high-value)
  3. Layer 3 - Anti-Persuasion: Detects 30%+ deviation from expected agent behavior patterns
  4. Layer 4 - Audit Authority: Immutable logging of all operations for post-hoc verification
